Auto-Stitching Photo Mosaics

Overview

In this project, we construct panoramic photo mosaics by stitching together overlapping photographs. We begin by manually selecting point correspondences to recover homographies and warp images into alignment, blending them with a Laplacian stack. We then automate the entire pipeline using Harris corner detection, adaptive non-maximal suppression for spatially uniform feature selection, descriptor matching, and RANSAC for robust homography estimation.

Recovering Homographies

First, we need to reconstruct the transformation between each pair of images. This transformation is called a homography, of the form $p'=Hp$, where $H$ is a $3 \times 3$ matrix with 8 degrees of freedom. We will recover the homography with a set of $(p',p)$ pairs of corresponding points selected from the two images. Four point correspondences are needed at minimum to produce a solution, but this might yield unstable results, so we instead select extra points and then solve for the least squares parameters of the homography matrix. Blending Images into a Mosaic

The left images in a given scene is warped to the right image via perspective transformation, and appropriate image boundaries are computed with respect to the original corners. The overall image boundaries are determined by the total range of all warped corners. Images are placed from left to right onto the output canvas based on the offset between the current image and the output's top-left corner. The Laplacian stack method implemented in the second project is used to blend the overlapping regions. After the images are blended, we apply some manual cropping on the output to eliminate the borders that do not contain any visual data. Harris Detector and Adaptive Non-Maximal Supression

We use the Harris corner detector algorithm to find a broad initial set of potential interest points. This involves computing Gaussian derivatives at each pixel, finding the second moment matrix in a Gaussian window around each pixel, and applying the corner response function. Then, we enforce a threshold to ensure that only sufficiently large corner responses are maintained. Finally, we perform adaptive non-maximal supression to obtain a fixed 500 points per image which are relatively evenly distributed spatially. Feature Descriptor Extraction

For each interest point determined by the output of the adaptive non-maximal supression, we take a $40 \times 40$ window around it, and downsample to a more general $8 \times 8$ patch. This patch is normalized to have zero mean and unit variance, then flattened into a vector, providing us with a feature descriptor for the interest point.

Feature Matching

To select correspondences between two sets of feature descriptors, we calculate the first and second nearest neighbors of each feature in one set with respect to all features in the other set according to the sum of squared differences metric. However, we want to ensure that the first nearest neighbor is a significantly better match than the second nearest neighbor. Hence, we only consider a correspondence a proper feature match if the ratio of errors between the first and second nearest neighbors is less than $0.5$ (i.e. the first nearest neighbor is at least twice as good as the second).
